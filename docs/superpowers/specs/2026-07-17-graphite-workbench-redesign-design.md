# 火宝短剧石墨工作台视觉改版

## 目标

在不改变现有功能、路由和六步制作流程的前提下，将当前偏橙色、装饰性较强的“电影控制台”界面改造成简约、克制、适合长时间使用的深色专业编辑工作台。

本次选定的唯一视觉方向为方案 2“石墨编辑台”。设计参照图：

## 设计原则

- 信息优先：使用留白、分隔线和字重组织层级，不依赖装饰、渐变或大面积强调色。
- 克制用色：蓝色只用于当前步骤、主要操作、选中态和进度；普通内容保持中性灰阶。
- 扁平稳定：去除发光、玻璃拟态、橙色渐变、暖金色和戏剧化阴影。
- 高密度但不拥挤：保留工作台所需的信息量，以一致的行高、间距和右侧检查器提升扫描效率。
- 功能不漂移：所有已有按钮、状态、表单、任务和数据关系保持原样，只重组视觉层级和响应式布局。

## 视觉系统

### 颜色

全局样式以语义变量统一管理，页面内不再新增零散的橙色和半透明暖色值。

| 用途 | 色值 |
| --- | --- |
| 应用底色 | `#15171A` |
| 一级表面 | `#1C1F23` |
| 悬停/选中表面 | `#24282D` |
| 分隔线/边框 | `#30343A` |
| 主要文字 | `#F4F5F6` |
| 次要文字 | `#9AA1AA` |
| 弱化文字 | `#6F7680` |
| 品牌与主要操作 | `#4C8DFF` |
| 品牌悬停 | `#6AA1FF` |
| 成功 | `#63A87A` |
| 警告 | `#D6A45E` |
| 错误 | `#C96B6B` |

状态不能只靠颜色表达，继续配合状态文字、图标或进度数字。警告色仅用于真实警告，不作为装饰色回流到主界面。

### 字体与层级

- 界面字体使用 `Inter, "Noto Sans SC", system-ui, sans-serif`。
- 移除界面中的衬线标题和全大写装饰字。
- 页面标题为 18–20px、600；区域标题为 14–16px、600；正文为 14px；辅助信息为 12–13px。
- 主要文字与次要文字通过字重和灰阶区分，避免过多字号。

### 形状、边框与阴影

- 常规控件和卡片圆角统一为 6px；标签和小状态为 4px。
- 区域之间优先使用 1px `#30343A` 分隔线，不使用浮夸卡片阴影。
- 弹层仅保留轻微边界阴影，用于和背景分离；常驻区域无阴影。
- 按钮使用纯色或描边，不使用渐变、发光和内高光。

### 交互状态

- 悬停：背景变为 `#24282D`，不改变布局尺寸。
- 选中：使用 1px 蓝色边框、浅蓝背景或左侧蓝色指示条，单个区域只采用一种方式。
- 键盘聚焦：保留清晰的 2px 蓝色外轮廓，不能只依赖浏览器默认样式被覆盖。
- 禁用：降低文字和边框对比度，同时保留可识别的控件轮廓。
- 常规按钮高度不低于 36px；图标按钮点击区域不小于 36×36px。

## 页面结构

### 全局框架

- 顶栏保持约 56px 高，左侧保留火宝品牌标识，中部展示当前项目与集数上下文，右侧保留已有全局操作。
- 删除胶片条、暖色光晕和纯装饰图形；顶栏使用 `#1C1F23` 与底部 1px 分隔线。
- 沿用项目已有的 `lucide-vue-next` 图标，不引入新图标库，也不新增手绘 SVG。

### 项目首页

- 保留当前项目创建、搜索、筛选、卡片点击和项目进入逻辑。
- 项目内容从强装饰卡片收敛为平整列表/网格：缩略图、项目名、集数进度、更新时间和主要入口形成明确顺序。
- 页面背景不再使用径向渐变；创建项目入口使用单一蓝色主按钮。
- 当前工作区中尚未提交的首页路由交互修改视为用户改动，实施时必须保留。

### 项目详情

- 项目信息与分集列表采用中性表面和细分隔线，不用大面积描边卡片套卡片。
- 当前分集、已完成、生成中、失败等状态使用统一状态圆点和文字。
- 新建分集、编辑项目和进入工作台维持现有行为，只调整按钮层级和密度。

### 六步工作台

- 左侧流程栏约 220px，保持“原始内容 → AI 改写 → 角色与场景 → 分镜拆分 → 视频生成 → 拼接导出”的现有顺序与状态逻辑。
- 当前步骤使用蓝色指示，已完成步骤使用低饱和绿色状态，未开始步骤保持灰色。
- 主工作区负责当前步骤的列表、编辑和预览；默认使用平整表格/行列表，不将每条记录包装成厚重独立卡片。
- 视频生成页以参照图为标准：中间为分镜任务列表，右侧约 320px 为当前分镜检查器，展示提示词、参考场景、参考角色、参数和任务状态。
- 选中任务行使用清晰蓝色边界；进度直接展示百分比；底部批量生成、重试失败和查看分镜维持现有功能与优先级。
- 原始内容、AI 改写、角色与场景、分镜拆分和拼接导出沿用同一“流程栏 + 主区 + 必要时右侧检查器”的视觉语言，不强行新增不存在的业务面板。

### 设置页

- 保留现有配置分组、服务商选择、表单编辑和测试行为。
- 左侧设置导航与右侧配置区使用平面分栏；表单输入统一底色、边框、聚焦态和错误态。
- 服务配置列表减少彩色徽章和卡片嵌套，主要操作使用蓝色，危险操作使用低饱和红色。

## 响应式规则

- `>= 1280px`：流程栏、主区和右侧检查器同时展示。
- `960–1279px`：流程栏保持，右侧检查器改为可收起面板或覆盖抽屉。
- `< 960px`：流程栏折叠为顶部步骤入口，主区单列，右侧检查器使用全宽抽屉；不得产生横向页面滚动。
- 表格在窄屏优先保留名称、状态和主要操作，次要字段进入展开详情。

## 实施边界

本次仅调整前端视觉和布局，主要落点为：

- `frontend/app/assets/studio.css`：重写全局语义色、字体、按钮、表单和表面样式。
- `frontend/app/layouts/default.vue`：简化顶栏和全局框架。
- `frontend/app/pages/index.vue`：项目首页视觉整理，保留现有未提交交互改动。
- `frontend/app/pages/drama/[id]/index.vue`：项目详情和分集列表视觉整理。
- `frontend/app/pages/drama/[id]/episode/[episodeNumber].vue`：六步工作台、视频任务列表和检查器布局整理。
- `frontend/app/pages/settings.vue`：设置导航、配置列表和表单视觉统一。

实施遵循最小改动：优先调整共享变量和现有样式，只有重复结构明确需要复用时才抽组件；不新增 UI 框架、图标库或状态管理依赖。

## 非目标

- 不修改后端接口、数据库结构、模型提供商或生成任务协议。
- 不改变六步工作流、生产门禁、角色/场景提取、分镜拆分或视频生成行为。
- 不新增登录、协作、通知、素材管理等产品能力。
- 不修改当前 `docs/prototype/` 中用户未提交的原型文件。
- 不制作新的影视插画或占位图；场景、角色和视频缩略图继续使用运行时真实素材。

## 验收标准

- 参照图中的石墨灰阶、蓝色强调、低阴影和紧凑信息层级在视频生成工作台中得到一致实现。
- 全站不再出现橙色主视觉、橙色渐变、暖金色装饰、胶片条、玻璃拟态或衬线界面标题。
- 所有现有主要操作仍可触达，任务状态、错误信息和键盘焦点清晰可辨。
- 在 1440×1024 视口下，工作台主列表和右侧检查器无裁切；在 960px 以下无页面级横向滚动。
- 新增一个结构性回归测试，约束核心主题色、字体、禁用渐变/发光及关键布局标记。
- 通过 `node --test tests/*.test.mjs` 和 `npm run build`。
- 使用同一 1440×1024 视口分别截取参照图与实现图进行并排视觉核对，修正明显的间距、边框、字号、圆角和层级偏差后再交付。
